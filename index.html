低 Codex 消耗工作流

Codex 写任务单，OpenCode/DeepSeek 改代码。

这个 skill 把 Codex 压缩成轻量调度器：它只负责明确目标、边界和安全规则，然后启动 worker。仓库阅读、代码修改和实现细节交给 OpenCode 的配置模型。

开始使用 查看仓库
少读Codex 不做大范围项目分析，不预先替 worker 设计实现。
少跑Codex 不复核 diff，不跑测试、构建、浏览器或修复循环。
可换默认 DeepSeek V4 Pro，也可以通过配置切换任何 OpenCode 模型。
可控用户保留最终验证和所有 Git 决策。

快速开始

安装 skill 和 OpenCode agent 后，在任意 Git 仓库里显式要求 Codex 使用这个 workflow。

Step 1

安装 skill

把仓库克隆到 Codex skills 目录，Codex 就能识别 `$codex-opencode-deepseek-workflow`。

Step 2

复制 agent

把 `codex-worker.md` 放到 OpenCode agents 目录，用来定义 worker 权限和行为。

Step 3

显式触发

普通 Codex 编程不会自动接管。只有你明确要求 OpenCode/DeepSeek 执行时才会启动。

PowerShell 安装命令 
git clone https://github.com/ysj98/codex-opencode-deepseek-workflow.git `
  "$HOME\.codex\skills\codex-opencode-deepseek-workflow"

New-Item -ItemType Directory -Force "$HOME\.config\opencode\agents" | Out-Null

Copy-Item `
  "$HOME\.codex\skills\codex-opencode-deepseek-workflow\opencode\agents\codex-worker.md" `
  "$HOME\.config\opencode\agents\codex-worker.md" `
  -Force

工作流

Codex 只把任务交出去，之后由用户做人工验收。这个边界是降低消耗的关键。

1

用户提需求

明确要使用 OpenCode/DeepSeek 或此 skill。

2

写任务单

Codex 只写目标、范围、约束和验收标准。

3

启动 worker

OpenCode 在当前 Git 工作区执行修改。

4

人工核查

用户查看 diff，并运行项目验证。

5

手动提交

只有用户确认后才执行 Git 操作。

模型配置

`codex-worker` 不绑定模型。脚本按参数、环境变量和 `worker.config.json` 解析最终 OpenCode 模型。

默认

DeepSeek V4 Pro

`deepseek-v4-pro` profile 指向 `deepseek/deepseek-v4-pro`。

临时

命令覆盖

传入 `-Model` 或设置 `CODEX_OPENCODE_MODEL`。

长期

配置 profile

新增 `modelProfiles` 条目，并修改 `defaultModelProfile`。

worker.config.json
{
  "defaultModelProfile": "deepseek-v4-pro",
  "modelProfiles": {
    "deepseek-v4-pro": {
      "model": "deepseek/deepseek-v4-pro"
    }
  },
  "agent": "codex-worker",
  "runsRoot": ""
}

边界很清楚

这个 workflow 不是完整自动交付链路。它是一个低消耗委托器。

Codex 做
  • 确认 Git 仓库
  • 生成短任务单
  • 调用 OpenCode worker
  • 报告日志和退出码
Worker 做
  • 按需读取项目文件
  • 实现代码修改
  • 保持在当前工作区内
  • 遇到不安全需求时停止说明
用户做
  • 查看 `git diff`
  • 运行测试和验证
  • 确认业务正确性
  • 决定是否提交 Git
Codex OpenCode Worker Workflow: 轻量委托，worker 执行，人工验收。
