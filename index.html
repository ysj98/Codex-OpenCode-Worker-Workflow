Codex OpenCode Worker Workflow

Codex 负责编排、任务单、diff 审查和验证验收。
OpenCode worker 调用配置模型执行代码修改。
用户保留运行确认和最终 Git 决策权。

开始使用 查看模型配置
0 Git 默认不 add、commit、push
1 diff 交付未提交变更供审查
2 轮 建议最多自动返修两轮

三步开始

安装 skill，复制 OpenCode agent，然后在任意干净的 Git 项目中显式调用。

Step 1

安装 skill

把仓库放入 Codex 用户技能目录，让 Codex 能发现 $codex-opencode-deepseek-workflow。

Step 2

复制 agent

把 codex-worker.md 复制到 OpenCode agents 目录。它只定义权限和行为，不绑定模型。

Step 3

显式触发

普通 Codex 编码不会自动接管。只有你明确要求该 workflow 时才会调用 OpenCode。

PowerShell install
git clone https://github.com/ysj98/codex-opencode-deepseek-workflow.git `
  "$HOME\.codex\skills\codex-opencode-deepseek-workflow"

New-Item -ItemType Directory -Force "$HOME\.config\opencode\agents" | Out-Null

Copy-Item `
  "$HOME\.codex\skills\codex-opencode-deepseek-workflow\opencode\agents\codex-worker.md" `
  "$HOME\.config\opencode\agents\codex-worker.md" `
  -Force

模型由配置决定

codex-worker 不写死模型。脚本会把解析出的模型通过 opencode run --model 传给 OpenCode。

默认 profile deepseek-v4-pro，指向 deepseek/deepseek-v4-pro。
临时覆盖 传 -Model "provider/model-id" 或设置 CODEX_OPENCODE_MODEL。
长期切换 新增 modelProfiles 条目，并修改 defaultModelProfile。
worker.config.json
{
  "defaultModelProfile": "deepseek-v4-pro",
  "modelProfiles": {
    "deepseek-v4-pro": {
      "model": "deepseek/deepseek-v4-pro"
    }
  },
  "agent": "codex-worker",
  "runsRoot": "",
  "worktreesRoot": ""
}

从需求到可审查 diff

Codex 做编排和验收，OpenCode worker 做执行，用户做最终确认。

1

分析项目

Codex 读取 AGENTS、README、manifest、脚本和相关源码。

2

写任务单

明确目标、规则、范围、禁止事项、验收标准和验证命令。

3

执行修改

OpenCode worker 在外部 worktree 中留下未提交 diff。

4

审查返修

Codex 审查 diff、运行验证；必要时输出返修意见并再次调用 worker。

Safety

默认不接管 Git 决策

工具只产出可审查 diff，不自动提交、不推送、不创建 PR。源工作区不干净时会停止，避免覆盖用户本地修改。

  • 任务单、日志、审查意见和执行摘要默认保存到用户级目录。
  • OpenCode worker 只在隔离 worktree 中编辑代码。
  • codex-worker 禁止 shell、子任务、外部目录、提交、推送和建 PR。
  • API Key 仍由 OpenCode 管理，本工具不读取、不保存、不打印。
Codex OpenCode Worker Workflow. Explicit delegation, reviewable diffs, human-owned Git decisions.
