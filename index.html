角色边界先说清楚

让计划、执行、验收各自独立，减少越权修改、隐式提交和“自己审自己”的风险。

Codex

架构与任务控制

读取项目规则，分析需求，输出《AI 开发任务单》，审查 diff，并生成返修意见。

DeepSeek V4

代码执行

通过 OpenCode 在隔离 worktree 中按任务单改代码，根据审查意见做最小返修。

User

运行与提交

运行项目，确认 UI 与业务效果，最终决定何时提交、推送或创建 PR。

从需求到可审查 diff

工具不替你做最终 Git 决策，只产出更容易审查、验证和返修的代码变更。

1

分析需求

Codex 读取 AGENTS.md、README、manifest、脚本和相关源码。

2

生成任务单

明确修改范围、禁止事项、实现要求、验收标准和建议验证命令。

3

执行修改

OpenCode 调用 DeepSeek V4，在外部 worktree 中留下未提交 diff。

4

审查返修

Codex 审查 git diff，必要时让 DeepSeek 按审查意见返修。

使用方式

推荐从 Codex 显式触发。你也可以手动生成任务单，再调用 OpenCode worker。

1在 Codex 中显式触发

普通 Codex 编码不会自动接管。只有你明确要求使用该 workflow 时，Codex 才会生成任务单并调用 OpenCode。

2让 DeepSeek 按任务单执行

脚本会创建外部 worktree，并通过 opencode run --agent codex-worker 调用 DeepSeek V4。

3由 Codex 审查 diff

Codex 检查变更范围、项目规则和验证结果。需要返修时，再把审查意见交给 DeepSeek。

手动调用 
powershell -NoProfile -ExecutionPolicy Bypass `
  -File "$HOME\.codex\skills\codex-opencode-deepseek-workflow\scripts\run-opencode-worker.ps1" `
  -RepoPath "D:\path\to\your-repo" `
  -TaskFile "C:\path\to\AI-DEV-TASK.md" `
  -TaskSlug "feature-name"
Safety

默认安全边界

适用于所有 Git 项目：工具不直接污染当前工作区，也不会接管最终发布流程。 codex-worker 只是一个可选 agent，不会影响你直接使用 OpenCode。

  • 源工作区不干净时拒绝启动 worker。
  • OpenCode 在外部 Git worktree 中执行代码修改。
  • DeepSeek worker 禁止提交、推送、建 PR 和运行 shell。
  • 只有显式调用 --agent codex-worker 时，该 agent 才会生效。
  • Codex 只审查 diff 和运行验证，最终提交由用户决定。

快速安装

把仓库克隆到 Codex 用户技能目录，再复制 OpenCode worker agent。

PowerShell 
git clone https://github.com/ysj98/codex-opencode-deepseek-workflow.git `
  "$HOME\.codex\skills\codex-opencode-deepseek-workflow"

New-Item -ItemType Directory -Force "$HOME\.config\opencode\agents" | Out-Null

Copy-Item `
  "$HOME\.codex\skills\codex-opencode-deepseek-workflow\opencode\agents\codex-worker.md" `
  "$HOME\.config\opencode\agents\codex-worker.md" `
  -Force
打开仓库 回到顶部
