快速开始

安装后，在任意 Git 项目中对 Codex 说「使用 OpenCode + DeepSeek 执行」即可。

Step 1

安装 skill

克隆仓库到 Codex 技能目录，Codex 即可发现该 workflow。

Step 2

复制 agent

将 codex-worker.md 复制到 OpenCode agents 目录，定义权限和行为。

Step 3

显式触发

普通编码不会自动接管。只有你明确要求该 workflow 时才会调用 OpenCode。

PowerShell 安装命令 
git clone https://github.com/ysj98/codex-opencode-deepseek-workflow.git `
  "$HOME\.codex\skills\codex-opencode-deepseek-workflow"

New-Item -ItemType Directory -Force "$HOME\.config\opencode\agents" | Out-Null

Copy-Item `
  "$HOME\.codex\skills\codex-opencode-deepseek-workflow\opencode\agents\codex-worker.md" `
  "$HOME\.config\opencode\agents\codex-worker.md" `
  -Force

工作流

从需求到修改只需四步，人工核查是必须环节。

轻量编排

Codex 避免大范围读代码，只提炼目标、边界和约束。

写任务单

明确目标、允许范围、禁止事项、验收标准和建议验证命令。

执行修改

OpenCode worker 在当前 Git 工作区留下未提交修改。

人工核查

查看 git diff，运行项目和测试，再决定是否提交。

模型配置

codex-worker 不绑定模型。修改 worker.config.json 即可切换。

默认deepseek-v4-pro 指向 deepseek/deepseek-v4-pro
临时传 -Model 参数或设 CODEX_OPENCODE_MODEL
长期新增 modelProfiles 条目，改 defaultModelProfile
worker.config.json
{
  "defaultModelProfile": "deepseek-v4-pro",
  "modelProfiles": {
    "deepseek-v4-pro": {
      "model": "deepseek/deepseek-v4-pro"
    }
  },
  "agent": "codex-worker",
  "runsRoot": ""
}

核心优势

为什么选择这个 workflow，而不是让 Codex 直接完成全流程。

Token 消耗显著降低

Codex 只做轻量编排，长上下文阅读和代码修改交给 OpenCode。不复核 diff、不二次修复。

模型完全解耦

agent 不绑定模型——切模型改一个 config 即可，脚本和 skill 无需任何修改。

零污染 · 可追溯

产物全部存 ~/.codex/runs/，业务仓库只留代码修改。每次运行生成结构化 JSON。

默认不接管 Git 决策

不自动 add / commit / push，不创建 PR。Worker 仅 4 个工具，禁止 shell、子任务、外部目录。API Key 由 OpenCode 管理，本工具不读取。任务单和日志存用户级目录，不污染业务仓库。

通用性极强

只要求 Git 仓库，不预设语言、框架、目录结构。不要求工作区干净。

用户保留最终决策权

Codex 不管 diff，Worker 不能 commit，全部 Git 操作由你手动执行。
